# PR7 
Отчет по практической работе: Отладка консольных приложений в MS Visual Studio
Данный репозиторий содержит результаты выполнения практических заданий по отладке кода на языке C#.

Цель работы: освоение инструментов отладки среды Microsoft Visual Studio и закрепление навыков документирования кода (XML-комментарии).

Используемые методы отладки
В ходе работы в среде Microsoft Visual Studio были активно использованы следующие инструменты:

Точки останова (Breakpoints): установка контрольных точек для приостановки выполнения программы и анализа текущего состояния памяти.

Пошаговое выполнение:

F10 (Step Over): проход по строкам кода без захода внутрь функций.

F11 (Step Into): детальный анализ логики внутри методов (использовалось для отладки метода Fibonacci и конструктора GType).

Окно «Локальные» (Locals): мониторинг значений переменных в текущем стеке вызовов в реальном времени.

Data Tips (Всплывающие подсказки): проверка значений переменных и содержимого массивов (например, letters) при наведении курсора мыши в режиме паузы.

Окно «Стек вызовов» (Call Stack): анализ последовательности вызовов методов для понимания иерархии выполнения приложения «Галактики».

Окно вывода (Output): просмотр результатов работы программы и системных сообщений.

Описание выполненных задач
1. Приложение «Фибоначчи»
Проблема: некорректный расчет последовательности из-за логической ошибки в итерациях и отсутствии обработки начальных значений.

Исправление: пересмотрена логика накопления суммы в цикле for и добавлена тернарная проверка для корректного возврата значения при n = 0.

Документирование: добавлены XML-комментарии (<summary>, <returns>) к методу Fibonacci.

2. Приложение «Галактики»
Проблема: логические ошибки в классификации типов галактик внутри конструктора класса GType (некорректное сопоставление данных).

Исправление: реализован оператор switch для точного сопоставления входных символьных кодов (S, E, I, L) с элементами внутреннего перечисления Type.

Документирование: классы Galaxy, GType и основные методы снабжены подробными XML-описаниями.

3. Приложение «Буквы»
Проблема: необходимость отладки работы с массивом символов и строковой конкатенации внутри цикла.

Исправление: проведена проверка границ цикла for и логики наполнения строки name, чтобы исключить ошибки при обращении к индексам массива letters.

Документирование: добавлены XML-комментарии для классов и методов вывода сообщений.
