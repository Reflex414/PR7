# PR7 
Отчет по практической работе: ОТЛАДКА ПРОГРАММЫ РАЗЛИЧНЫМИ СПОСОБАМИ

1. Приложение «Фибоначчи»
Для этого кода основным методом была проверка логики вычислений внутри цикла:

Step Into (F11): Использовался для входа в метод Fibonacci и детального отслеживания того, как меняются переменные n1, n2 и sum на каждой итерации.

Окно Locals (Локальные переменные): Позволило заметить, что при определенных входных данных итерации цикла или начальные значения переменных приводили к неверному результату (ошибка «на единицу»).

Точка останова (Breakpoint): Была установлена на начале цикла for, чтобы проверить состояние переменных перед первой итерацией.

2. Приложение «Галактики»
Здесь отладка была сфокусирована на обработке данных и конструкторах:

Step Over (F10): Использовался в методе IterateThroughList для быстрого прохода по списку объектов и проверки, все ли элементы создаются без исключений.

Data Tips (Всплывающие подсказки): При наведении на объект theGalaxy во время паузы проверялось, корректно ли свойство MyGType приняло значение из перечисления (именно здесь была обнаружена ошибка инициализации в конструкторе).

Call Stack (Стек вызовов): Помог отследить цепочку от вызова IterateThroughList до конструктора GType, чтобы понять, на каком этапе теряются данные о типе галактики.

3. Приложение «Буквы»
В данном коде акцент был сделан на работе с массивами и строками:

Breakpoint с условием: Установка точки останова внутри цикла for, чтобы проверить значение переменной name на конкретном шаге (например, когда i = 5).

Проверка границ массива: С помощью инструментов отладки проверялось соответствие индекса i длине массива letters, чтобы убедиться в отсутствии ошибки IndexOutOfRangeException.

Визуализатор строк: Использовался для контроля того, как посимвольно собирается строка name из массива char[] при каждом вызове метода SendMessage.арии для классов и методов вывода сообщений.
