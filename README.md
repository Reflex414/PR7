# Отчет по практической работе №7: Отладка программы различными способами

## 1. Приложение «Фибоначчи»

### Методы отладки
- **Step Into (F11)** – детальное отслеживание переменных `n1`, `n2` и `sum` внутри метода.
- **Окно Locals** – выявление ошибки «на единицу» в итерациях цикла.
- **Точка останова (Breakpoint)** – проверка состояния системы перед входом в цикл.

### Изменения в коде
- Исправлено условие цикла  
  `for (int i = 2; i <= n; i++)`  
  для корректного расчета n-го числа.
- Добавлена тернарная проверка  
  `return n == 0 ? n1 : n2;`  
  для обработки случая, когда запрашивается нулевое число Фибоначчи (ранее могло приводить к логической ошибке).
- Добавлены XML-комментарии для документирования входных параметров и возвращаемого значения метода.

## 2. Приложение «Галактики»

### Методы отладки
- **Step Over (F10)** – быстрый проход по списку объектов в `IterateThroughList`.
- **Data Tips** – обнаружение пустых значений в свойстве `MyGType` при наведении курсора.
- **Call Stack** – анализ передачи данных от инициализации списка до конструктора `GType`.

### Изменения в коде
- Полностью переработан конструктор класса `GType`: реализован оператор `switch`, который сопоставляет входной символ (например, `'S'`) с соответствующим элементом перечисления `Type.Spiral`.
- Исправлена ошибка, из-за которой свойство `MyGType` оставалось неинициализированным (`null`).
- Добавлены XML-комментарии для классов `Galaxy`, `GType` и метода `Main` для обеспечения читаемости кода.

## 3. Приложение «Буквы»

### Методы отладки
- **Условная точка останова** – проверка переменной `name` на конкретной итерации цикла.
- **Проверка границ массива** – контроль индекса `i` относительно `letters.Length`.
- **Визуализатор строк** – инспекция процесса сборки строки из массива символов.

### Изменения в коде
В данном приложении алгоритм изначально был корректен. Основные изменения коснулись документирования кода:
- добавлены XML-комментарии для всех методов и классов;
- проведена проверка работы цикла через условные точки останова для подтверждения корректности сборки строки;
- внедрены документирующие комментарии, описывающие назначение параметров `name` и `msg` в методе `SendMessage`.
